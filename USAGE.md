# 日常工作管理智能体使用指南

## 快速开始

### 1. 切换到工作目录
```bash
cd /workspace/工作
```

### 2. 启动管理智能体会话
使用明确的会话标识符，例如：
```
session_id="ses_work_20260212"
```

### 3. 加载管理相关技能
建议加载的技能：
- `playwright`：浏览器自动化
- `dev-browser`：网页交互
- `git-master`：版本控制

### 4. 执行日常管理任务
- 项目管理
- 文档处理
- 自动化脚本
- 浏览器操作

## 智能体切割要点

### 严格隔离
- **论文智能体**：只在`/workspace/论文/`目录工作
- **管理智能体**：只在`/workspace/工作/`目录工作
- 使用不同的`session_id`前缀区分

### 会话管理
```
论文会话：ses_paper_YYYYMMDD
工作会话：ses_work_YYYYMMDD
```

### 技能选择
| 任务类型 | 推荐技能 | 示例 |
|---------|---------|------|
| 网页自动化 | playwright, dev-browser | 数据采集、表单填写 |
| 文档处理 | 写作技能 | 报告生成、文档整理 |
| 任务管理 | 快速任务技能 | Todo列表、进度跟踪 |
| 版本控制 | git-master | 提交、分支、合并 |

## 验证切割效果

运行测试脚本：
```bash
./scripts/test_cutting.sh
```

## 常见工作流

### 项目管理
1. 在`projects/`目录创建项目文件夹
2. 使用Git进行版本控制
3. 创建任务列表和文档

### 文档处理
1. 在`docs/`目录存放文档
2. 使用Markdown格式
3. 定期备份重要文件

### 自动化脚本
1. 在`scripts/`目录存放脚本
2. 添加执行权限：`chmod +x script.sh`
3. 测试脚本功能

## 故障排除

### 问题：会话混淆
**解决方案**：
- 明确指定`session_id`
- 检查当前工作目录
- 重新启动清晰会话

### 问题：目录越界
**解决方案**：
- 确认当前在正确目录：`pwd`
- 使用绝对路径时检查权限
- 遵循隔离原则

### 问题：技能冲突
**解决方案**：
- 按需加载技能，不加载不必要的技能
- 明确指定`load_skills`参数
- 使用合适的`category`

## 最佳实践

1. **会话清晰**：每次启动时明确会话标识
2. **目录专注**：一个会话只在一个目录工作
3. **技能精简**：只加载必要的技能
4. **版本控制**：定期提交重要更改
5. **测试验证**：定期运行切割测试
