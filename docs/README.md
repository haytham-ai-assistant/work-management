# 工作区文档管理系统

## 目录结构说明

```
docs/
├── projects/                    # 项目文档
│   ├── 视触技术/               # 视触技术项目
│   │   ├── README.md           # 项目说明
│   │   └── [待上传文档]        # 项目相关文档
│   ├── requirements/           # 需求文档
│   └── specifications/         # 技术规范文档
├── processes/                  # 流程文档
├── templates/                  # 模板文件
└── references/                 # 参考资料
```

## 文档格式建议
1. **PDF格式** - 用于正式文档、合同、规范等（保持原始格式）
2. **Markdown格式** - 用于技术文档、说明、README等（适合Git版本控制）
3. **纯文本格式** - 用于配置文件、脚本说明等

## 上传指南
1. 将文档分类放入相应目录
2. 为每个文档创建简短的描述文件（README.md）
3. 定期提交到Git仓库进行版本控制
4. 避免上传大型二进制文件到Git仓库
